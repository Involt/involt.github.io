Triggering functions

Sometimes, exchanging the data is not enough. In more complex projects there are events that occurs from both sides, especially triggering some event in arduino from html button. It's great to know that Involt cares about simplicity of such events.

Triggering from app to device

For example, use HTML element to send its value and trigger some event in Arduino sketch:

<div class="ard button P6 value-255 fn-involt">Click me</div>
...

void loop(){

involtReceive();


if(fname == "involt"){

doSomething...

};


fname = ""; //clear the name to prevent from repeating the function each time

}
...

There are other possibilites to do this in app. The first is to use involt-cta with fn HTML attribute. It will only send the function when pressed. This element can be used along with Involt pagination and involt-pagelink to trigger an event when user goes to another page.

<div class="involt-cta" fn="involt">Click me</div>

JQuery method to send function:

$(this).sendFn("involt");

Pure JS solution:

involt.arduinoFn("involt");

Triggering from device to app

Adding this function to sketch will send name of function to app.

involtSendFunction("involt");

This one is tricky. Unlike previous versions and listening for changes in interval, now it's done with declaring the function in involtFunction JS object. Include the custom script in new file and attach it to end of HTML head section.

involtFunction.involt = function(){

doSomething...

};
Involt is ready to use Chrome packaged app template. You can use it to quickly create interactive object with software to hardware communication. The goal is to help designers to overcome the obstacles at prototyping stage.
